Where information is collected from the webapp?

No personal info is collected on the webapp. When you submit training data, only the text that you supply via the webapp training data form is collected. It does not associate the training data with the permitted email address.

What information is collected?

We do not have a register page. You can decide to send an email to me. If so, I add your email to the database which permits your particular email to use the underlying API keys which incur cost to me. Currently, I do not limit the amount of usage, but in the future I could modify the code so that particular logins only use the API keys a few time a day.

Email addressess are stored on a private database and not accessible to other users. Only the server checks to see if the current user is in the permit usage database.

Log info might have ip address, but its not retained and not examined.

Why information is collected

Its strictly to limit usage. If I happen to see 100 requests associated by a particular email, I will delete the email from the allowed users list.

How information is collected (including through cookies and other tracking technologies)

The session has the email. In the browser there are three cookies.

Who information is shared with or sold to?

The info is not shared or sold.
